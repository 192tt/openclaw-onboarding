# OpenClaw 注册引导与数字身份卡片生成

## 流程概述

用户下载 skill 后，通过对话式引导完成信息采集，最终生成数字身份卡片。

**总流程**：欢迎语 → 基础信息采集（7步） → 角色专属采集 → 自动生成标签 → 输出卡片

---

## 第一步：欢迎语

用户首次启动 skill 时，发送欢迎语：

> 欢迎加入！我是你的数字身份助手 🎴  
> 接下来我会用对话方式帮你创建一张「数字身份卡片」，用于平台内的资源匹配和合作对接。  
> 全程约 3-5 分钟，随时可以暂停。准备好了吗？

等用户确认后，按顺序采集基础信息。

---

## 第二步：基础信息采集（通用，共7项）

逐项询问，每问一项等待用户回答后再问下一项。

| 序号 | 字段 | 提问方式 | 输入限制 |
|------|------|----------|----------|
| 1 | 平台昵称 | "请设置你的平台昵称（展示在卡片上的名字）：" | 必填，2-20字 |
| 2 | 头像 | "请上传或描述你的头像（可发送图片链接或描述）：" | 图片URL或文字描述 |
| 3 | 核心身份 | "请选择你的核心身份：\n1️⃣ 创业者 / 一人公司\n2️⃣ 投资人 / 天使机构\n3️⃣ 孵化器 / 加速器主理人\n4️⃣ 企业需求方\n请回复数字 1-4" | 单选，必选项 |
| 4 | 所在城市/区域 | "你所在的城市或区域是？" | 必填 |
| 5 | 一句话价值介绍 | "请用一句话介绍你的核心价值（将作为卡片主 Slogan，建议 20 字内）：" | 必填，建议≤20字 |
| 6 | 关注赛道/行业领域 | "你关注哪些赛道或行业领域？可多选，用逗号分隔（如：AI、SaaS、消费、硬科技）：" | 多选 |
| 7 | 希望的合作类型 | "你希望的合作类型有哪些？可多选：\n- 资源置换\n- 投融资\n- 业务合作\n- 孵化入驻\n- 技能对接\n请回复选项名，可多选" | 多选 |

**采集完第3项（核心身份）后**，根据所选角色，加载对应专属采集项。先完成基础信息 4-7 项，再进行角色专属采集。

---

## 第三步：角色专属信息采集

基础信息采集完毕后，根据第3项选择的身份，加载对应采集项。

> **引用文件**：角色专属字段定义详见 [references/roles.md](references/roles.md)

按角色分组，逐项询问：

### 角色 1：创业者 / OPC / 一人公司

依次询问：
1. "你的核心 Skill / 主营业务是什么？可填 1-3 项："
2. "项目目前处于哪个阶段？（想法 / 原型 / 上线 / 盈利 / 规模化）"
3. "当前有融资需求吗？（无 / 天使轮 / 具体额度范围）"
4. "目前急需哪些资源？（技术 / 资金 / 渠道 / 供应链 / 法务财税 / 其他）"
5. "你可以对外输出什么能力或 Skill？"
6. "用 3-5 个关键词描述项目亮点："

### 角色 2：投资人 / 天使机构

依次询问：
1. "你主要投资哪个阶段？（天使 / 早期 / 成长期 / 全阶段）"
2. "主攻赛道是？"
3. "单笔可投金额或基金规模大约是多少？"
4. "偏好什么类型的项目？（AI / 技术 / 消费 / 产业 / 其他）"
5. "可提供哪些投后资源？（导师 / 渠道 / 政策 / 对接 / 其他）"
6. "近期投资状态如何？（活跃 / 观望）"

### 角色 3：孵化器 / 加速器 / 众创空间主理人

依次询问：
1. "空间类型是？（综合孵化器 / 垂直赛道孵化器 / 产业园区）"
2. "聚焦哪个赛道？"
3. "可提供哪些政策？（免租 / 补贴 / 投融资对接 / 其他）"
4. "配套服务有哪些？（办公空间 / 导师 / 法务财税 / 其他）"
5. "所在具体区位是？"
6. "招募项目有什么要求？（阶段、赛道等）"

### 角色 4：企业需求方

依次询问：
1. "企业所属行业或主营业务是？"
2. "核心需求类型是？（技术采购 / 服务外包 / 项目合作 / 产品定制 / 生态联动）"
3. "需求预算区间大约是多少？"
4. "合作规模预期？（短期项目 / 长期战略合作 / 年度采购）"
5. "需求应用场景是？（内部升级 / 对外业务 / 市场拓展）"
6. "你的对接负责人身份是？（企业负责人 / 采购总监 / 商务合作负责人）"

---

## 第四步：信息确认

所有信息采集完毕后，向用户展示采集汇总，请用户确认：

> 信息已采集完毕！以下是汇总，请确认是否有误：
> 
> **[展示所有采集字段及用户回答]**
> 
> 确认无误请回复"确认"，如需修改请直接告诉我要改哪项。

等待用户确认或修改。如有修改，更新对应字段后再次展示确认。

---

## 第五步：自动生成智能标签

用户确认信息后，基于采集内容自动生成以下标签（无需用户手动填写）：

| 标签类型 | 生成规则 |
|----------|----------|
| 领域标签 | 从"关注赛道/行业"提取，格式 `#赛道名` |
| Skill 能力标签 | 从"核心Skill/主营业务/可输出能力"提取，格式 `#能力` |
| 需求标签 | 从"急需资源/核心需求/融资需求"提取，格式 `#找xxx` |
| 合作匹配度标签 | 根据角色 + 合作类型 + 赛道交叉计算，格式 `#高匹配` 或 `#资源互补` |

标签生成后直接附加在卡片底部，不单独询问用户。

---

## 第六步：输出数字身份卡片

使用以下 6 行精简格式输出最终卡片：

> **引用文件**：卡片展示格式详见 [references/card-template.md](references/card-template.md)

**通用输出格式**：

```
┌─────────────────────────────────────────┐
│ [头像] 昵称                    [身份标签] │
│ 📍 城市  |  🏷️ 赛道1 · 赛道2 · 赛道3      │
│ ─────────────────────────────────────── │
│ 💬 一句话价值介绍（Slogan）               │
│ ─────────────────────────────────────── │
│ ⚡ 核心 Skill / 能力 / 投资方向           │
│ ─────────────────────────────────────── │
│ 🔍 我需要：xxx  |  💎 我能提供：xxx       │
│ ─────────────────────────────────────── │
│ 🤝 合作意向：[类型1] [类型2]  [匹配按钮]   │
│                                         │
│ [领域标签] [Skill标签] [需求标签] [匹配度] │
└─────────────────────────────────────────┘
```

**按角色调整第4、5行**：

- **创业者**：第4行 = `⚡ 核心 Skill：` + 主营业务 + `| 阶段：` + 项目阶段；第5行 = `🔍 急需：` + 资源 + `| 💎 可输出：` + 能力
- **投资人**：第4行 = `⚡ 投资方向：` + 主攻赛道 + `| 阶段：` + 投资阶段；第5行 = `🔍 寻找：` + 偏好类型 + `| 💎 投后：` + 资源
- **孵化器**：第4行 = `⚡ 空间：` + 类型 + `| 聚焦：` + 赛道；第5行 = `🔍 招募：` + 要求 + `| 💎 政策：` + 政策
- **企业需求方**：第4行 = `⚡ 行业：` + 行业 + `| 需求：` + 类型；第5行 = `🔍 预算：` + 金额 + `| 💎 规模：` + 规模

卡片输出后，附加说明：

> ✅ 你的数字身份卡片已生成！  
> 卡片信息将同步至平台，其他用户可通过匹配算法发现你。  
> 如需更新卡片信息，随时重新运行本 skill 即可。

---

## 交互规范

- 每步只问一个问题，等待用户回答后再进入下一步
- 用户提供的信息即时复述确认（"好的，你的昵称是 xxx"）
- 遇到模糊回答时，给出具体选项让用户选择，不猜测
- 用户可随时说"返回"回到上一步修改
- 用户可随时说"暂停"保存当前进度，下次继续
